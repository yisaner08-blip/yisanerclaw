# Python 编码风格

## 通用规范

- 缩进：4 空格，禁止 Tab
- 行宽：≤ 100 字符
- 引号：字符串用双引号，单字符用单引号
- 导入：标准库 → 第三方 → 本地，每组间空行

## 简洁原则（ponytail 懒惰阶梯增强）

- 🚨 写代码前先检查 ponytail 懒惰阶梯（7 级）：已有方案？标准库？一行？→ 见 [`ponytail.md`](ponytail.md)
- 能用一行代码搞定就不用两行，避免不必要的中间变量
- 优先用推导式、三元表达式、链式调用精简代码
- 删除优于添加，朴素优于花哨，文件越少越好
- 没有明确需求前不做提前抽象（YAGNI）
- 用 `ponytail:` 注释标记有意的简化

## 注释规范

- 🚨 每个函数、类声明处必须有中文注释说明用途
- 🚨 变量声明（尤其非显而易见者）必须有中文注释
- 🚨 关键逻辑、分支判断处必须有中文行内注释
- 注释紧贴代码上方，与代码同行不换行

## 类型注解

- 公开函数必须有类型注解
- 使用 `list[dict]` 而非 `List[Dict]`（Python 3.10+）

## 错误处理

- 工具函数返回字符串错误信息，不抛异常
- ⚠️ 避免裸 `except Exception: pass`，至少记录错误
- 外部调用（API、文件、网络）必须 try/except

## 可靠性

- 回答基于可靠信息（文档、实测结果、代码验证）
- ⚠️ 不确定时明确说明并给出验证方法
- 不做没有依据的推测
